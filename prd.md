# 智能报表 AI Agent - 产品需求文档 (PRD)

## 1. 产品定位

**智能报表 Agent** 是一款基于多 Agent 协作的智能化报表生成平台。用户通过定义章节（text描述 + 工具链），AI 按录制好的流程稳定执行，生成符合预期的报告。

**核心理念**：像编程一样定义报告生成流程，录制-重放机制确保输出稳定。

---

## 2. 功能需求分析

### 2.1 模板与章节

#### 模板结构
```
模板A
├── 章节1：概述
│   ├── text: "总结本周安全运营总体态势..."
│   └── 工具链: [tool1, tool2, ...]
├── 章节2：威胁事件分析
│   ├── text: "最近一周威胁事件，按类型/来源/严重程度维度..."
│   └── 工具链: [tool_query_sql, tool_gen_table, tool_analyze]
└── 章节3：漏洞分析
    ├── text: "新增漏洞分布，按严重等级/类型维度..."
    └── 工具链: [tool_query_sql, tool_gen_pie_chart, tool_summarize]
```

#### 章节属性
| 属性 | 说明 |
|-----|------|
| `text` | 用户感兴趣的描述，类似 text2sql 的 prompt，供 AI 理解用户意图 |
| `工具链` | 该章节使用的工具列表（含参数），录制保存，下次直接重放 |

#### text 描述示例
```
"最近一周的威胁事件有哪些，按类型/来源/严重程度维度统计数据，
包含表格展示，最后有分析总结"

"本周新增漏洞分布，按严重等级和类型维度，用饼图展示"

"统计被攻击的Top10资产，按攻击次数排序"
```

#### 时间参数动态化
- text 中如包含"最近一周"、"上周"、"本月"等时间描述
- 执行时自动转换为具体时间范围（如 2026-03-26 ~ 2026-04-02）
- 工具链参数中的时间字段引用 text 中的时间语义

### 2.2 探索式调试模式（预构建块）

#### 痛点
- 完整流程太长（text → 数据 → 图表 → 分析 → 文档）
- 用户无法预览中间结果，不满意需要重新跑整个流程
- 调试效率低，影响体验

#### 解决方案：预构建块（Prebuilt Block）

用户提供一个 text，AI 执行并返回结果（数据或图表），用户可以：
1. 继续调整 text，重新执行
2. 满意后，保存整个编排流程（text + 工具链）为一个"预构建块"
3. 在章节中引用预构建块，直接使用已验证的流程

#### 用户操作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    探索式调试界面                            │
│                                                             │
│  调试过程（用户看到的是结果卡片）:                            │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 结果A: 威胁类型分布表格              [结果不满意]    │   │
│  │ ┌────────────────────────────┐                      │   │
│  │ │ 类型        数量           │                      │   │
│  │ │ 恶意软件    125           │                      │   │
│  │ │ 钓鱼攻击    89            │                      │   │
│  │ └────────────────────────────┘                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 结果B: 被攻击资产表格 + 分析         [结果满意 ✓]   │   │
│  │ ┌────────────────────────────┐                      │   │
│  │ │ 资产      攻击次数         │                      │   │
│  │ │ asset1    1523            │                      │   │
│  │ │ asset2    987             │                      │   │
│  │ └────────────────────────────┘                      │   │
│  │ 分析: 资产被攻击Top5以Web服务为主...                │   │
│  │                                                     │   │
│  │ [保存为预构建块]                                     │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

#### 预构建块保存机制

**问题**：调试过程可能产生多次工具调用，用户只想保存最终满意的结果。

**解决**：用户只需要说"这个结果好"，系统自动追溯依赖链。

**执行记录格式**：
```json
{
  "call_id": "call_007",
  "tool": "gen_table",
  "input_refs": ["call_005"],      // 引用了 call_005 的输出
  "output_ref": "output_007",     // 本次输出ID，供后续调用引用
  "output_summary": "表格: 6列10行",
  "result_visible": true            // 是否展示给用户
}
```

**依赖追溯保存流程**：
1. 用户点击"保存" → 标记该结果为满意
2. 系统根据 `input_refs` 追溯完整的依赖链
3. 按依赖顺序自动排好工具链
4. 用户只需命名，即可保存

**示例**：
```
调试过程发生了10次调用，用户只满意"结果B"（被攻击资产表格+分析）

系统追溯：
结果B ← call_009(summarize) ← call_007(gen_table) ← call_005(query_sql)

自动构建工具链：
1. query_sql(查资产信息)
2. gen_table(生成资产表格)
3. summarize(分析总结)
4. gen_markdown(生成章节)

用户命名："被攻击资产分析"
保存完成
```

#### 预构建块结构
```json
{
  "id": "block_001",
  "name": "威胁类型分布分析",
  "description": "查询威胁事件类型分布，返回表格",
  "text": "最近一周威胁事件，按类型分布统计",
  "tools": [
    {
      "tool": "query_sql",
      "params": {
        "sql": "SELECT type, count(*) FROM threats WHERE time_range='{time_range}' GROUP BY type",
        "time_param": "recent_week"
      }
    },
    {
      "tool": "gen_table",
      "params": {
        "data_source": "query_sql_result",
        "columns": ["类型", "数量"]
      }
    }
  ],
  "created_by": "user_001",
  "created_at": "2026-04-02"
}
```

#### 预构建块在章节中的引用
```json
{
  "id": "ch2",
  "title": "威胁事件分析",
  "blocks": [
    {"ref": "block_001"},           // 引用已保存的预构建块
    {"ref": "block_003"}            // 再引用另一个预构建块
  ],
  "text": "最近一周的威胁事件分析，参考预构建块",
  "tools": []                        // 也可以直接定义工具链，与 blocks 二选一
}
```

#### 两种模式对比

| 模式 | 适用场景 | 说明 |
|-----|---------|------|
| **直接编辑** | 模板已有清晰结构 | 直接在章节中编辑 text + 工具链 |
| **探索调试** | 不确定效果，需反复调试 | 先在预构建块中调试，满意后引用 |

#### 预构建块管理
- 预构建块库：用户个人保存的预构建块列表
- 公共库：企业内共享的预构建块
- 版本管理：支持更新已引用的预构建块

### 2.3 工具库设计

#### 工具分类
| 工具类型 | 示例工具 | 用途 |
|---------|---------|------|
| 查询工具 | `query_sql` | 执行 SQL 查询数据 |
| 查询工具 | `query_api` | 调用外部 API 获取数据 |
| 查询工具 | `query_log` | 查询日志数据 |
| 图表工具 | `gen_table` | 生成数据表格 |
| 图表工具 | `gen_pie_chart` | 生成饼图 |
| 图表工具 | `gen_line_chart` | 生成折线图 |
| 图表工具 | `gen_bar_chart` | 生成柱状图 |
| 分析工具 | `analyze_trend` | 趋势分析 |
| 分析工具 | `analyze_anomaly` | 异常检测 |
| 文档工具 | `summarize` | 文字总结生成 |
| 文档工具 | `gen_markdown` | 生成 Markdown 章节 |
| 文档工具 | `gen_image` | 生成图片章节 |

#### 工具配置
每个工具可配置给不同的 **Sub-Agent**：
- **查询 Agent**：负责数据查询相关工具
- **图表 Agent**：负责图表生成相关工具
- **分析 Agent**：负责数据分析相关工具
- **文档 Agent**：负责文档组装相关工具

#### 工具链录制格式
```json
{
  "chapter_id": "ch2",
  "tools": [
    {
      "tool": "query_sql",
      "params": {
        "sql": "SELECT type, count(*) FROM threats WHERE time_range='{time_range}' GROUP BY type",
        "time_param": "recent_week"
      }
    },
    {
      "tool": "gen_table",
      "params": {
        "data_source": "query_sql_result",
        "columns": ["类型", "数量"]
      }
    },
    {
      "tool": "summarize",
      "params": {
        "input": "基于{gen_table_result}分析威胁事件类型分布特点"
      }
    }
  ]
}
```

### 2.5 录制-重放机制

#### 录制过程
1. 用户定义/调整模板的 text 描述
2. 首次执行时，AI 按意图调用工具
3. 系统记录：工具名 + 参数（时间参数用变量占位）
4. 保存到模板的章节工具链中

#### 重放过程
1. 用户选择模板，点击"生成报告"
2. 主 Agent 按章节顺序读取工具链
3. 时间变量自动替换为实际时间
4. 依次调用工具，输出稳定结果

#### 局部调整
- 只调整某章节的 text 或工具链 → 只影响该章节
- 其他章节不受影响

## 4. 技术架构

### 4.1 核心模块

```
┌─────────────────────────────────────────────────────────────┐
│                      用户交互层                              │
│   (Web 控制台 / API / 钉钉机器人)                            │
├─────────────────────────────────────────────────────────────┤
│                      主 Agent                                │
│   - 模板解析                                                  │
│   - 章节编排                                                  │
│   - Sub-Agent 调度                                          │
│   - 报告组装                                                  │
├─────────────────────────────────────────────────────────────┤
│                    Sub-Agent 层                              │
│   ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│   │ 查询 Agent │ │ 图表 Agent │ │ 分析 Agent │ │ 文档 Agent │      │
│   └──────────┘ └──────────┘ └──────────┘ └──────────┘      │
├─────────────────────────────────────────────────────────────┤
│                      工具层                                  │
│   query_sql │ query_api │ gen_table │ gen_chart │ summarize │
├─────────────────────────────────────────────────────────────┤
│                      数据层                                  │
│   安全日志 │ 告警数据 │ 资产数据 │ 外部 API                    │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 执行流程

```
用户: 点击"生成报告"
           │
           ▼
主 Agent: 加载模板，解析章节列表
           │
           ▼
For each 章节:
    │
    ├── 解析 text 中的时间语义（"最近一周" → 时间范围）
    │
    ├── 按工具链顺序执行:
    │   ├── Sub-Agent 调度
    │   ├── 调用工具
    │   └── 录制工具调用结果
    │
    └── 输出章节内容
           │
           ▼
主 Agent: 组装所有章节为完整报告
           │
           ▼
返回给用户 / 推送 / 保存
```

| 风险 | 影响 | 应对 |
