## Context

当前安全运营报表生成依赖人工编写，效率低且难以保证一致性。安全运营团队需要能够通过自然语言快速获取数据、生成图表、进行分析，并生成符合行业规范的报表。

系统特点：
- 用户基数：安全运营工程师
- 数据源：大数据平台提供多种查询接口（按数据域分类）
- 行业特点：网安行业术语多，报表格式相对标准化

## Goals / Non-Goals

**Goals:**
- 支持用户通过自然语言prompt查询数据
- 支持数据可视化（表格、柱状图、折线图、饼图等）
- 支持对数据进行归纳、总结、分析
- 提供报表模板系统，内置网安行业标准模板
- 支持单Data Part调试，用户可以预览和调整结果
- 保存验证过的prompt组合为预制件，可作为few-shot示例
- 通过语义匹配将用户prompt路由到正确的查询工具

**Non-Goals:**
- 不实现Text2SQL直接转换（采用工具注册表+预制件匹配方式）
- 不支持用户自定义图表类型（只支持预定义的图表格式）
- 不实现多人协作编辑（单用户版本）

## Decisions

### 1. 工具注册表 + 预制件混合检索

**决策**：采用工具注册表语义检索 + 预制件优先匹配的双层策略

```
用户prompt → 预制件检索 → 匹配成功 → 直接使用
                    ↓ 匹配失败
              工具注册表检索 → 匹配成功 → 生成新预制件
                    ↓ 匹配失败
              返回"未找到合适工具"
```

**理由**：
- 工具注册表保持系统级纯净，不被用户调试数据污染
- 预制件积累用户验证过的模式，提升匹配准确度
- 避免工具注册表别名无限膨胀

### 2. Prompt改写Agent职责

**决策**：改写Agent负责将用户prompt拆解为"n个查询调用 + m个分析调用"

改写结果结构：
```json
{
  "data_parts": [
    {
      "id": "dp_001",
      "type": "query",
      "original_prompt": "最近一周哪些资产被攻击",
      "rewritten_prompt": "查询被攻击资产，按攻击次数Top10",
      "tool_id": "asset_attacks",
      "params": { "time_range": "last_week", "limit": 10 }
    },
    {
      "id": "dp_002",
      "type": "analyze",
      "original_prompt": "基于上面数据分析",
      "depends_on": ["dp_001"],
      "rewritten_prompt": "分析资产被攻击的趋势和特征"
    }
  ]
}
```

**理由**：
- 查询和分析分离，便于单独调试
- 保留原始prompt用于知识积累
- 依赖关系显式声明，执行顺序清晰

### 3. 语义向量化方案

**决策**：采用通用Embedding模型（OpenAI text-embedding-3 或开源sentence-transformers）

**理由**：
- 网安行业术语与通用Embedding有一定兼容性
- 预制件匹配作为二次校验，提升准确度
- MVP阶段先用通用Embedding，后续可切换到领域微调模型

### 4. 预制件结构

**决策**：预制件包含三要素
- `original_prompt`: 用户原始输入
- `rewritten_prompt`: 改写后的标准prompt（作为别名）
- `tool_chain`: 工具ID + 参数 + 格式化方式

**理由**：
- original_prompt保留用户意图，可用于未来智能匹配
- rewritten_prompt作为标准别名，供后续用户参考
- tool_chain确保执行路径可复现

### 5. 报表模板结构

**决策**：模板为JSON格式，按章节（Chapter）和数据部分（Data Part）组织

```json
{
  "id": "template_001",
  "name": "安全周报",
  "chapters": [
    {
      "id": "ch1",
      "title": "概述",
      "data_parts": ["dp_001"]
    },
    {
      "id": "ch2",
      "title": "威胁分析",
      "data_parts": ["dp_002", "dp_003"]
    }
  ]
}
```

**理由**：
- 章节对应报告的结构层次
- Data Part可跨章节复用
- 支持模板编辑和版本管理

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 语义匹配准确度不足 | 用户prompt找不到合适工具 | 预制件二次校验；提供手动工具选择 |
| 改写Agent理解偏差 | 拆解出错误的查询/分析组合 | 调试模式允许用户调整；录制-重放机制 |
| 图表生成格式单一 | 无法满足个性化需求 | 预定义多种图表类型；支持ECharts配置 |
| 预制件膨胀 | 用户难以管理 | 预制件分类（个人/公共）；搜索和筛选功能 |
